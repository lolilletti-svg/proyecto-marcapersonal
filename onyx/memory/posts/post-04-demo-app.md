## Versión completa (English)

This is what the AP reconciliation process looks like when it runs automatically.

[VIDEO/GIF: app in action]

Input: a vendor statement in PDF or Excel.
Output: a reconciliation report — matched invoices, unmatched items, discrepancies flagged.

What it handles automatically:
- Parses PDF statements without manual conversion
- Normalizes invoice numbers across formats (leading zeros, dashes, prefixes)
- Consolidates the same vendor across multiple locations
- Flags mismatches and exports a clean result

What used to take me 2+ hours per vendor now runs in under a minute.

I built this with Claude API, JavaScript, and HTML — no prior development experience.
The hardest part wasn't the technology. It was writing down every step of the process
with enough precision to translate it into logic.

That's the skill I'm now applying to every manual process I touch.
